# Написание Unit тестов. Лабораторная работа №4

## 1. Цели и задачи тестирования

**Цель:** Обеспечить проверку правильности работы математических функций, вычисляющих площадь и периметр прямоугольника.

**Задачи:**
- Протестировать точность вычисления площади при различных входных параметрах
- Протестировать точность вычисления периметра при разных входных значениях
- Проверить обработку крайних случаев (нулевые параметры)
- Протестировать обработку недопустимых входных данных (отрицательные числа)
- Организовать автоматизированный процесс выполнения тестов

## 2. Описание тестируемого продукта

**Продукт:** Библиотека для вычисления геометрических параметров прямоугольника

**Основной функционал:**
- area(a, b) - вычисление площади прямоугольника
- perimeter(a, b) - вычисление периметра прямоугольника

**Технические требования:**
- Функции должны поддерживать работу с целыми и вещественными числами
- Должна быть реализована обработка невалидных входных данных
- Результаты вычислений должны соответствовать математическим формулам

## 3. Область тестирования

**Включаемые компоненты:**
- Функция area(a, b) - вычисление площади
- Функция perimeter(a, b) - вычисление периметра
- Обработка стандартных входных данных
- Обработка предельных значений
- Обработка ошибочных данных

**Исключаемые компоненты:**
- Графические элементы интерфейса (не предусмотрены)
- Интеграция с базами данных (не используется)
- Сетевые подключения (не реализованы)

## 4. Стратегия тестирования

**Методология:** Модульное тестирование (Unit-тестирование)

**Подход к тестированию:** Тестирование по методу черного ящика

**Направления тестирования:**
- Проверка функциональности
- Тестирование предельных значений
- Проверка обработки исключительных ситуаций
- Позитивное тестирование (корректные данные)
- Негативное тестирование (некорректные данные)

**Реализация:**
- Применение фреймворка unittest
- Разработка независимых тестов для каждой функции
- Классификация тестов по функциональным блокам

## 5. Критерии приемки

**Условия успешного тестирования:**
- Все разработанные unit-тесты выполняются успешно
- Обеспечено полное покрытие тестируемого кода
- Корректно обрабатываются все предельные случаи
- Ошибки валидации входных данных обрабатываются надлежащим образом
- Код тестов соответствует стандартам PEP8

**Показатели качества:**
- Удобочитаемость и сопровождаемость тестового кода
- Независимость тестовых сценариев
- Наличие информативных сообщений об ошибках
- Корректное использование проверочных утверждений

## 6. Ожидаемые результаты

**Результаты тестирования:**
- Отчет о прохождении тестов с процентным показателем успешных тестов
- Детальная информация о неудачных тестах (при наличии)
- Показатели покрытия кода тестами
- Предложения по устранению выявленных недочетов

**Выходная документация:**
- Протокол тестирования с результатами выполнения каждого теста
- Описание выявленных дефектов с последовательностью воспроизведения
- Вывод о степени готовности продукта к эксплуатации
